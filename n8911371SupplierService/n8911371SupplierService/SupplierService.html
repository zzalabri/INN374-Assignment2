6 public method descriptions

1. bool IsExistComponentNo(string componentNo)
   Function Description : To check the componentNo which exists in the Supplier table.
            If return true, this means the componentNo exists in the Supplier table.
            If return false, the means the componentNo does not exist in the Supplier table.
			
2. string GetComponentName(String componentNo)
   Function Description : To get one ComponentName record from the Supplier table while the input data is the componentNo.
            If no record, the return value is the empty string.
        
3. int GetCost(String componentNo)
   Function Description : To get one Cost record from the Supplier table while the input data is the componentNo.
            If no record, the return value is 0.
        
4. int GetAmount(String componentNo)
   Function Description : To get the current Amount record from the Supplier table while the input data is the componentNo.
            If no record, the return value is 0.
        
5. bool CheckAmount(string componentNo, int exceptOrderAmount)
   Function Description : To check the current amount from the Supplier table which can offer enough amount for the special componentNo.
            If return true, this means the supplier can provide the componentNo with the number of exceptOrderAmount.
            If return false, the means the supplier can provide the componentNo with the number of exceptOrderAmount and return value is 0.
        
6. bool OrderAmount(string componentNo, int orderAmount)
   Function Description : To get the amount from the Supplier table with the specific componentNo and the number of orderAmount.
            If return true, the order is successful to order the number of orderAmount with the specific componentNo.
            If return false, the order fails to order the number of orderAmount with the specific componentNo.
        
Constraints

1. The length of the componentNo must be 5, such as 100000. If the value is empty or the null value, each method will not get any record and the return value will be  the empty string or 0 based on different methods.
        
2. The number of exceptOrderAmount and orderAmount must be over 0. If below 1, the CheckAmount and OrderAmount methods will return the false value.
